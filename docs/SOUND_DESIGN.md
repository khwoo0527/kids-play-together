# 🎵 풍선 터트리기 게임 — 사운드 디자인 문서

## 📌 역할 지정
당신은 **모바일 캐주얼 게임 전문 사운드 디자이너**입니다.
Nintendo, Supercell(Clash of Clans), King(Candy Crush) 수준의 게임 오디오를 설계한 경험을 가지고 있으며,
Web Audio API를 사용한 **코드 기반 사운드 합성(Procedural Audio)** 에 능숙합니다.

---

## 🎮 게임 컨텍스트

| 항목 | 내용 |
|------|------|
| 게임 장르 | 캐주얼 터치 게임 (풍선 터트리기) |
| 대상 연령 | 초등학생 6~13세 |
| 플레이 환경 | 터치 모니터, 태블릿, 스마트폰 브라우저 |
| 분위기 | 밝고 신나고 귀엽고 통통 튀는 느낌 |
| 기술 스택 | 순수 Web Audio API (외부 파일 없음, 코드로 합성) |

---

## 🔊 필요한 사운드 목록

### 1. BGM (배경음악)
**요구사항:**
- 초등학생이 질리지 않고 들을 수 있는 **밝고 경쾌한 루프 음악**
- 한 루프가 **최소 40초 이상** — 반복감이 느껴지지 않아야 함
- 볼륨이 크지 않고 게임 플레이를 방해하지 않는 **배경 레벨**
- 텐션이 유지되어야 함 — 졸리거나 심심한 느낌 금지
- **딩동딩동 단순 멜로디 반복 절대 금지** — 어지럽고 거슬림
- 참고 레퍼런스: Candy Crush BGM, 마리오 카트 BGM 느낌
- Web Audio API `OscillatorNode` + `BufferSource` + `BiquadFilterNode` 등 활용

**구현 방식 (코드 예시 포함):**
```js
// _startBGM() 메서드 내부에 구현
// 아래 요소들을 조합하여 구성할 것:
// - 킥드럼: OscillatorNode (sine, 160→40Hz 피치드롭)
// - 하이햇: BufferSource (화이트노이즈 + highpass filter)
// - 베이스라인: OscillatorNode (sine, 낮은 음역대)
// - 패드: OscillatorNode (sine, 부드러운 코드 지속음)
// - 멜로디 악센트: 드문드문, 너무 규칙적이지 않게
```

---

### 2. 풍선 팝 소리 (pop)
**요구사항:**
- **통통 튀고 귀여운** 소리 — "펑!" 이 아닌 "뽁!" 느낌
- 콤보 숫자가 높아질수록 **음이 올라가야** 함 (보상감)
- 짧고 임팩트 있게 — 0.2초 이내
- 듣기 좋고 중독성 있어야 함 (계속 터트리고 싶게)

```js
// pop(combo = 1) 메서드에 구현
// 현재 구현: OscillatorNode sine + triangle 조합
// 개선 포인트: 더 통통한 느낌을 위해 짧은 화이트노이즈 burst 추가 고려
```

---

### 3. 콤보 달성 소리 + 보이스 (combo)
**요구사항:**
- 3콤보 이상 달성 시 재생
- **징글 소리** (짧은 상승 멜로디) + **"나이스!" 보이스** 조합
- 보이스: **높고 귀여운 어린이 느낌** — 성인 여성 목소리 금지
- Web Speech API 사용 금지 (OS 의존, 품질 불균일)
- **Web Audio API로 직접 합성**: 비브라토 + 배음 조합으로 어린이 목소리 느낌
- 콤보 레벨 높을수록 더 화려한 징글

**보이스 합성 파라미터 (현재 기준):**
```js
// _synthVoice(word, startTime) 메서드
// - 기음: OscillatorNode (sine, 880~1760Hz 범위)
// - 배음 1: OscillatorNode (triangle, freq * 2.0)
// - 배음 2: OscillatorNode (sine, freq * 3.0)
// - 비브라토: LFO (OscillatorNode 6.5Hz → GainNode → frequency)
// - 음정 패턴: 단어별로 다르게 (나이스 = [880, 1318, 1046, 1760])
// 개선 포인트: 더 어린이스럽게, 더 귀엽게
```

---

### 4. 게임 시작 소리 (gameStart) — 미구현
**요구사항:**
- 카운트다운 3, 2, 1 각각 소리
- "시작!" 에 짧고 신나는 팡파레

---

### 5. 게임 종료 소리 (gameOver) — 미구현
**요구사항:**
- 결과 화면 등장 시 재생
- 점수에 따라 다른 소리 (잘했을 때 vs 보통일 때)
- 칭찬받는 느낌의 밝고 따뜻한 소리

## 🛠️ 현재 코드 구조

```
js/sound.js
├── class SoundManager
│   ├── init()              — 첫 터치 시 AudioContext 초기화
│   ├── pop(combo)          — 풍선 팝 소리
│   ├── combo(count)        — 콤보 징글 + 보이스
│   ├── toggleBGM()         — BGM 온/오프
│   ├── _ding(freq,t,dur)   — 단음 재생 유틸
│   ├── _synthVoice(word,t) — 합성 보이스
│   ├── _resumeForSFX()     — BGM 꺼진 상태에서도 SFX 재생
│   └── _startBGM()         — BGM 루프 시작
└── const sound = new SoundManager()
```

---

## ⚠️ 제약 조건

1. **외부 오디오 파일 없음** — 모든 소리는 Web Audio API로 합성
2. **첫 터치 후 AudioContext 초기화** — 브라우저 자동재생 정책 대응
3. **BGM 끄면 SFX는 계속 재생** — context suspend/resume 조합
4. **모바일 저사양 대응** — OscillatorNode 동시 최대 10개 이하 권장
5. **파일 위치**: `js/sound.js` 단일 파일 내 구현

## 💡 개선 요청사항

아래 항목들을 전문가 관점에서 재설계해주세요:

1. **BGM 전면 재설계**
   - 40초+ 루프, 텐션 유지, 딩동딩동 금지
   - 드럼 비트 + 패드 + 베이스 + 가끔 악센트 구조 권장

2. **팝 소리 개선**
   - 더 통통하고 중독적인 뽁! 소리

3. **합성 보이스 개선**
   - 진짜 어린이 목소리처럼 들리게
   - "나이스!" 한 단어를 귀엽고 선명하게

4. **전체 볼륨 밸런싱**
   - BGM : SFX : 보이스 = 1 : 2 : 1.5 정도 권장
